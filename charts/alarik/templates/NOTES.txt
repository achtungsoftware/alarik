SCALING

  Up:   kubectl --namespace {{ .Release.Namespace }} scale statefulset {{ include "alarik.fullname" . }} --replicas=N
        Safe at any time. New pods join, and existing nodes migrate a share of the data onto them
        automatically.

  Down: ./scale-down.sh {{ include "alarik.fullname" . }} N {{ .Release.Namespace }}
        Use the script, NOT `kubectl scale`. Removing a pod outright leaves the shards it held
        missing until a manual resync notices; the script drains each pod first so its data moves
        onto the remaining nodes before it goes away.

{{- if not .Values.persistence.enabled }}

WARNING: persistence is DISABLED. Pod storage is an emptyDir, so all data - objects, users,
buckets and access keys alike - is destroyed when a pod restarts. Never run this way outside a
throwaway test.
{{- end }}
